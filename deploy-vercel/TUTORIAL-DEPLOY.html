Pengantar — Kenapa Perlu Deploy?

TeleDrive bisa diakses dari semua device setelah di-deploy

Preview/sandbox di mana TeleDrive ini dibuat hanya berjalan sementara di container development. Saat tidak ada yang memproses, server mati dan URL tidak bisa diakses.

Untuk membuatnya bisa diakses kapan saja, dari device mana pun, kita perlu deploy (publish) ke server hosting yang aktif 24/7.

💡
Apa itu Deploy? Deploy = meng-upload kode aplikasi ke server hosting agar bisa diakses publik melalui URL permanen. Seperti upload video ke YouTube — setelah upload, semua orang bisa akses kapan saja.

Persiapan — Yang Perlu Disiapkan

Pastikan kamu sudah punya semua ini sebelum mulai:

  • Akun GitHub — Daftar gratis di github.com/signup
  • Akun Vercel — Daftar di vercel.com/signup (bisa login pakai akun GitHub)
  • Git terinstall di komputer — Download di git-scm.com/downloads
  • Bot Telegram + Chat ID — Lihat cara buat di bagian 3 & 4
✨
Sudah punya semua? Kalau kamu sudah punya bot token dan chat ID (karena sudah pakai TeleDrive sebelumnya), langsung loncat ke Langkah 5: Download File.

Buat Bot Telegram (Kalau Belum Punya)

Bot Telegram sebagai "penyimpanan" file kamu
LANGKAH 3.1

Buka BotFather di Telegram

Buka Telegram, cari @BotFather (punya centang biru ✅). Ketik /newbot.

↓
LANGKAH 3.2

Beri Nama Bot

Beri nama bot yang kamu suka, misalnya: "Drive Pribadi Saya"

↓
LANGKAH 3.3

Beri Username Bot

Username harus unik dan diakhiri "bot". Contoh: my_drive_12345_bot

↓
LANGKAH 3.4

Simpan Bot Token

BotFather akan kasih token seperti ini:

Contoh Token⚠️ JANGAN DIBAGIKAN
7123456789:AAHxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx

Simpan token ini! Kamu akan butuh saat setup TeleDrive.

⚠️
Keamanan Token Bot token adalah "kunci" akses ke bot kamu. Jangan pernah membagikannya ke orang lain. Kalau token bocor, kirim /revoke ke @BotFather untuk membuat token baru.
4

Dapatkan Chat ID

Chat ID adalah ID unik channel/chat kamu di Telegram. Semua file akan disimpan di chat ini.

LANGKAH 4.1

Buat Channel atau Group di Telegram

Opsi A: Buat Channel baru (ikon broadcast) — rekomendasi karena bisa upload banyak file tanpa batas anggota.
Opsi B: Gunakan Group — bisa kalau mau share akses dengan orang lain.
Opsi C: Gunakan chat pribadi dengan bot — kurang ideal karena tidak bisa pinned messages.

↓
LANGKAH 4.2

Tambahkan Bot ke Channel

Buka channel → klik nama channel → Administrators → Add Admin → cari username bot kamu → jadikan admin.

↓
LANGKAH 4.3

Dapatkan Chat ID

Cara paling mudah:

  1. Kirim pesan apapun di channel tersebut
  2. Buka browser, kunjungi: https://api.telegram.org/botTOKEN_KAMU/getUpdates
  3. Cari bagian "chat":{"id": -100xxxxxxxxxx}
  4. Angka negatif itu adalah Chat ID kamu

Contoh URL:

Browser
https://api.telegram.org/bot7123456789:AAHxxx/getUpdates

Contoh response:

JSON Response
{
  "ok": true,
  "result": [{
    "message": {
      "chat": {
        "id": -1001234567890,
        "title": "Drive Saya",
        "type": "channel"
      }
    }
  }]
}

Chat ID kamu: -1001234567890 (angka negatif)

💡
Chat ID selalu dimulai dengan -100 untuk channel. Kalau group, dimulai dengan -. Kalau chat pribadi, angka positif. Simpan Chat ID ini — kamu akan butuh saat setup TeleDrive.

Download File Deploy

File yang perlu di-deploy sudah disiapkan di folder deploy-vercel/. Ada 2 cara mendapatkannya:

Cara A: Download ZIP (Paling Mudah)

Download file deploy-vercel.zip (90KB) yang sudah tersedia. Extract di komputer kamu.

Cara B: Copy Manual

Struktur folder yang harus kamu punya:

Struktur Folder
deploy-vercel/
├── vercel.json              # Konfigurasi Vercel
├── package.json             # Info project
├── .gitignore               # Git ignore rules
├── api/
│   ├── tg-api.js            # Proxy JSON → Telegram
│   └── tg-upload.js         # Proxy upload file → Telegram
└── public/
    ├── teledrive.html       # Aplikasi utama (~436KB)
    ├── logo.svg             # Logo
    └── robots.txt           # SEO
Arsitektur: Browser → Vercel (Proxy) → Telegram API
6

Upload ke GitHub

LANGKAH 6.1

Buat Repository Baru di GitHub

1. Buka github.com/new
2. Repository name: teledrive (atau nama lain yang kamu suka)
3. Jangan centang "Add a README file"
4. Klik "Create repository"

↓
LANGKAH 6.2

Buka Terminal / Command Prompt

Navigasi ke folder deploy-vercel yang sudah di-extract:

Terminal
cd /path/ke/deploy-vercel
↓
LANGKAH 6.3

Init Git dan Push

Jalankan perintah-perintah ini satu per satu:

TerminalSalin dan jalankan satu per satu
# Inisialisasi git repository
git init

# Tambahkan semua file
git add -A

# Commit dengan pesan
git commit -m "TeleDrive siap deploy"

# Hubungkan ke GitHub (ganti USERNAME_KAMU)
git remote add origin https://github.com/USERNAME_KAMU/teledrive.git

# Set branch utama ke 'main'
git branch -M main

# Push ke GitHub
git push -u origin main
⚠️
Ganti USERNAME_KAMU dengan username GitHub kamu yang sebenarnya. Contoh: kalau URL GitHub kamu github.com/budi123, maka ganti jadi budi123.
↓
LANGKAH 6.4

Verifikasi di GitHub

Buka repository kamu di browser. Semua file sudah terupload ✅

7

Deploy ke Vercel

GitHub → Vercel: otomatis build dan deploy
LANGKAH 7.1

Login ke Vercel

Buka vercel.com/login dan login pakai akun GitHub.

↓
LANGKAH 7.2

Buat Project Baru

1. Di dashboard, klik "Add New..." → "Project"
2. Di bagian "Import Git Repository", cari dan pilih repository teledrive yang tadi dibuat
3. Klik "Import"

↓
LANGKAH 7.3

Konfigurasi Project

Isi konfigurasi berikut:

Setting Value
Project Name teledrive (atau terserah)
Framework Preset Other (penting!)
Root Directory Biarkan default (.)
Build Command Kosongkan / hapus
Output Directory Kosongkan / hapus
Install Command Kosongkan / hapus
🔴
PENTING: Framework Preset harus "Other"! Jangan pilih Next.js, React, atau apapun. Karena ini bukan project Next.js, melainkan static files + Edge Functions.
↓
LANGKAH 7.4

Klik Deploy!

Klik tombol biru "Deploy" dan tunggu ±30 detik.

↓
🎉 SELESAI!

Deploy Berhasil!

Vercel akan memberikan URL seperti:

URL TeleDrive Kamu
https://teledrive-abc123.vercel.app

Gunakan TeleDrive

LANGKAH 8.1

Buka URL TeleDrive

Buka URL Vercel kamu di browser (HP, laptop, tablet — apa saja). Contoh: https://teledrive-abc123.vercel.app

↓
LANGKAH 8.2

Isi Settings

1. Klik ikon ⚙️ Pengaturan (di sidebar bawah atau tombol settings)
2. Masukkan Bot Token dari @BotFather
3. Masukkan Chat ID (angka negatif -100xxx)
4. Klik Simpan

↓
LANGKAH 8.3

Mulai Pakai!

TeleDrive siap digunakan. Upload file, buat folder, kelola file — semua tersimpan di channel Telegram kamu dan bisa diakses dari device mana pun!

Custom Domain (Opsional)

Kalau kamu punya domain sendiri, bisa dihubungkan ke Vercel:

LANGKAH 9.1

Tambah Domain di Vercel

Dashboard Vercel → Project → Settings → Domains → Tambahkan domain kamu (misal: drive.contoh.com)

↓
LANGKAH 9.2

Setting DNS

Di DNS registrar (Cloudflare, Namecheap, dll), tambahkan:

TypeNameValue
CNAMEdrivecname.vercel-dns.com
↓
LANGKAH 9.3

Tunggu Propagasi

DNS perlu waktu ±5-30 menit untuk propagasi. Setelah itu, buka drive.contoh.com dan TeleDrive akan tampil!

💡
HTTPS otomatis! Vercel otomatis mengurus SSL certificate. Domain kamu akan langsung pakai HTTPS tanpa konfigurasi tambahan.

Cara Update TeleDrive

Kalau ada versi baru TeleDrive (file teledrive.html yang diperbarui):

TerminalDi folder deploy-vercel
# 1. Timpa file lama dengan yang baru
cp /path/teledrive-baru.html public/teledrive.html

# 2. Commit dan push
git add public/teledrive.html
git commit -m "Update TeleDrive ke versi terbaru"
git push
🔄
Auto-Deploy! Setiap kali kamu push ke GitHub, Vercel akan otomatis mendeteksi perubahan dan me-redeploy. Tidak perlu login ke Vercel sama sekali. Butuh ±30 detik.

Alternatif: Deploy ke Render / Railway

Kalau kamu butuh upload file sangat besar (>50MB) atau tidak mau pakai Vercel, ada folder deploy/teledrive-server/ yang berisi server Node.js mandiri:

Deploy ke Render.com

Langkah
# 1. Push deploy/teledrive-server/ ke GitHub
# 2. Buka render.com → New → Web Service
# 3. Pilih repository teledrive-server
# 4. Build Command: (kosongkan)
# 5. Start Command: node server.js
# 6. Klik Deploy — selesai!

Deploy ke Railway.app

Langkah
# 1. Push deploy/teledrive-server/ ke GitHub
# 2. Buka railway.app → New Project → Deploy from GitHub
# 3. Pilih repository teledrive-server
# 4. Railway otomatis detect Node.js dan deploy

Troubleshooting

MASALAH 1

Halaman 404 atau "Not Found" setelah deploy

Cause: Framework Preset tidak dipilih "Other"
Fix: Di Vercel → Project Settings → General → Framework Preset → ubah ke Other → Redeploy

MASALAH 2

Upload file gagal / error

Cause: File terlalu besar untuk Edge Runtime
Fix: Gunakan folder deploy/teledrive-server/ di Render/Railway (tidak ada batas ukuran)

MASALAH 3

"Bad Request: bot token is invalid"

Cause: Bot token salah atau sudah direvoke
Fix: Cek token di @BotFather → /mybots → API Token

MASALAH 4

"Bad Request: chat not found"

Cause: Chat ID salah atau bot belum ditambahkan ke channel
Fix: Pastikan bot sudah dijadikan admin di channel. Cek Chat ID dengan cara di bagian 4.

MASALAH 5

Git push gagal: "fatal: remote origin already exists"

Fix: Jalankan git remote remove origin lalu ulangi git remote add origin ...

MASALAH 6

Vercel deploy gagal: "Error: No output directory"

Fix: Pastikan Build Command dan Output Directory dikosongkan di settings project Vercel

Arsitektur & Keamanan

Bagaimana Cara Kerjanya?

Alur Data
Browser (HP/Laptop)
    │
    ├─ GET /teledrive.html  ──→  Vercel CDN (static file)
    │
    ├─ POST /api/tg-api     ──→  Vercel Edge Function
    │                              └─→ forward JSON ke Telegram API
    │                                 └─→ return response ke browser
    │
    └─ POST /api/tg-upload  ──→  Vercel Edge Function
                                   └─→ parse bot_token, chat_id
                                   └─→ forward raw body ke Telegram
                                      └─→ return response ke browser

Keamanan

  • Bot Token disimpan di localStorage browser kamu — TIDAK di server
  • Chat ID juga disimpan lokal di browser
  • Semua data file tetap di Telegram Cloud — server hanya sebagai proxy
  • Edge Functions tidak menyimpan log sensitif
  • HTTPS otomatis dari Vercel
  • Headers keamanan: X-Content-Type-Options, X-Frame-Options, Referrer-Policy
🔒
Privasi Maksimal Server proxy hanya meneruskan request tanpa menyimpan apapun. Bot token dan data kamu tetap di perangkat dan Telegram sendiri.
