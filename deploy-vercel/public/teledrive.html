API & Backup Configuration

Mendukung channel privat (ID numerik -100XXXXXXXXXX) maupun channel publik (username @namachannel). Bot harus menjadi admin channel dengan izin Kirim & Hapus Pesan.

Jika api.telegram.org diblokir (mis. jaringan kantor), deploy proxy gratis ke Cloudflare, Deno, Vercel, atau Render. Script siap-pakai: tg-proxy-worker.js, tg-proxy-deno.js, tg-proxy-vercel.js, tg-proxy-render.js. Semua request API & unduhan file akan diteruskan via proxy ini.

Klik salah satu proxy di atas untuk pakai cepat, atau tempel URL lain di kolom Proxy URL.

PROXY PUBLIK GRATIS (tanpa deploy sendiri)

Proxy publik berikut langsung jalan tanpa perlu deploy. Token bot akan terlihat oleh penyedia proxy, tapi untuk penggunaan pribadi risikonya rendah. Pilih salah satu:

⚠️ Batasan: corsproxy.io & proxy.cors.sh hanya support GET request (Tes Koneksi). Untuk Sync & Upload (POST), gunakan proxy milik sendiri (Cloudflare/Vercel/Render).

Panduan deploy cepat (2 menit, gratis)

Cloudflare Workers (limit 100rb/hari):

1. Buka workers.cloudflare.com → login → "Create Worker"

2. Hapus kode default, salin isi tg-proxy-worker.js ke editor

3. Klik "Deploy" → salin URL Worker (mis. https://namaproxy.username.workers.dev)

4. Tempel URL tersebut di kolom Proxy URL di atas, lalu Simpan

Render.com (gratis, tanpa limit request/hari):

1. Buat repo GitHub dengan 3 file: server.js (isi tg-proxy-render.js), package.json, render.yaml

2. Buka render.com → Sign up pakai GitHub → "New +" → "Web Service"

3. Pilih repo Anda → Instance Type: Free → "Create Web Service"

4. Salin URL (mis. https://teledrive-proxy-xxxx.onrender.com) → tempel di Proxy URL

Vercel (gratis, tanpa kartu kredit):

1. Buat repo GitHub baru (kosong, tanpa README)

2. Buat struktur folder: package.json (isi {"type":"module"}) + api/[...path].js

3. Isi api/[...path].js dengan script dari tg-proxy-vercel.js

4. Buka vercel.com → Import repo → klik "Deploy"

5. Salin URL + akhiran /api/ → klik tombol "Vercel" di atas

Deno Deploy (gratis, tanpa kartu kredit):

1. Buka dash.deno.com → Sign in pakai GitHub

2. Klik "New Playground" → hapus kode default → paste isi tg-proxy-deno.js

3. Klik "Save & Deploy" → salin URL (mis. https://xxxx.deno.dev) → tempel di Proxy URL

Unduh file HTML mandiri (1 file saja, tanpa server). Simpan di komputer Anda, buka dengan browser apa pun. Semua fitur berfungsi termasuk sync & proxy.

Sinkronisasi Channel

Fitur ini memindai SELURUH riwayat channel Anda untuk menemukan semua file. Mendukung channel privat (ID -100...) maupun channel publik (@username). Menggunakan teknik forward & delete transit yang aman — file tidak berpindah atau terduplikasi di channel. Kini dengan penanganan rate-limit otomatis agar tidak ada file yang terlewat.

Cara kerja & informasi lanjutan

1. Mencari ID Tertinggi: Sistem melakukan binary search (~24 panggilan API) untuk menemukan pesan terbaru di channel.

2. Memindai Satu per Satu: Dari ID tertinggi hingga ID 1, setiap pesan di-forward sementara ke channel yang sama, diperiksa apakah mengandung file, lalu dihapus. Jika terkena rate-limit, sistem otomatis menunggu & mencoba ulang (tidak ada file yang terlewat). Pesan dengan konten terproteksi (content protection) dicatat terpisah.

3. Deteksi Duplikat: File yang sudah ada di drive Anda (berdasarkan file_id Telegram) tidak akan diduplikasi.

4. Resume: Jika scan terhenti, Anda bisa melanjutkan dari posisi terakhir tanpa mengulang dari awal.

5. Estimasi Waktu: Channel dengan 10.000 pesan butuh ~17 menit (kecepatan Normal). Pesan yang sudah dihapus akan dilewati otomatis.

Scan sebelumnya terhenti. Terakhir dipindai: pesan ID -. Anda bisa melanjutkan atau mulai ulang.
